# Runvas

`runvas`는 Runvas의 문서, 백엔드, 모바일 앱을 함께 관리하는 단일 저장소입니다.

공통 기준은 `docs/`에서 관리하고, 실제 구현은 `backend/`와 `mobile/`에서 관리합니다.

## 프로젝트 개요

Runvas는 사용자가 지도 위에서 직접 러닝 코스를 설계하고 공유할 수 있는 위치 기반 러닝 코스 플랫폼입니다.

기존 러닝 서비스가 거리, 속도, 칼로리 같은 운동 기록 관리에 집중한다면, Runvas는 사용자가 하트, 별, 동물, 캐릭터 같은 형태의 코스를 만들고 공유하며 러닝 자체를 하나의 창의적인 경험으로 즐길 수 있게 하는 데 집중합니다.

## 저장소 구조

- `docs/`: 백엔드와 모바일이 함께 맞춰야 하는 공통 기준과 제품 범위
- `backend/`: 서버 API, 데이터 저장, 인증, GPX 응답 구현
- `mobile/`: 모바일 앱 화면, 지도 인터랙션, API 연동 구현

## 문서 역할

`docs/`에는 다음처럼 백엔드와 모바일이 함께 맞춰야 하는 내용을 둡니다.

- API 기준
- 데이터 모델
- 좌표계 및 경로 거리 기준
- 지도 범위 조회 규칙
- GPX 다운로드 규칙
- 커뮤니티 게시글, 댓글, 좋아요 규칙
- 에러 응답 형식
- 용어 정의

다음 내용은 각 구현 디렉토리에서 관리합니다.

- 화면 상세 UI 구현
- 서버 내부 아키텍처
- DB 마이그레이션 파일
- 배포 환경별 설정값
- 플랫폼별 라이브러리 사용법

## 팀 개발 설정

커밋 메시지 검증 hook을 사용하려면 저장소를 받은 뒤 한 번 실행합니다.

```bash
./scripts/setup-git-hooks.sh
```

Git 작업 규칙과 커밋 메시지 기준은 [협업 가이드](./docs/collaboration.md)를 따릅니다.

## 변경 원칙

공통 기준이 바뀌면 `backend/`와 `mobile/` 모두 영향을 받습니다.

따라서 아래 변경은 반드시 `docs/` 문서를 먼저 수정한 뒤 구현합니다.

- API 요청/응답 필드 추가, 삭제, 이름 변경
- 데이터 타입 변경
- 좌표 순서, 단위, 거리 계산 방식 변경
- 코스 공개 범위나 조회 조건 변경
- GPX 파일 구조 변경
