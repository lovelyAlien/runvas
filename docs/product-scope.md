# 제품 범위

## 목표

Runvas의 목표는 사용자가 러닝 코스를 직접 그리고, 저장하고, 공유하며, 다른 사용자가 만든 창의적인 러닝 코스를 탐색할 수 있게 하는 것입니다.

핵심 가치는 "얼마나 멀리, 얼마나 빠르게 달렸는가"가 아니라 "어떤 경험을 위해 달릴 것인가"입니다.

MVP는 코스 제작 도구와 커뮤니티를 함께 포함합니다.
사용자는 코스를 만들고, 공개하고, 게시글과 댓글로 경험을 나누며, 좋은 코스와 콘텐츠에 좋아요로 반응할 수 있습니다.

## 핵심 사용자 흐름

1. 사용자는 지도 위에서 러닝 코스를 직접 그립니다.
2. 모바일 앱은 경로 좌표를 기반으로 거리와 예상 소요 시간을 계산해 보여줍니다.
3. 사용자는 제목, 설명, 태그, 공개 여부를 입력해 코스를 저장합니다.
4. 다른 사용자는 현재 지도에서 보고 있는 지역 범위를 기준으로 주변 코스를 탐색합니다.
5. 사용자는 마음에 드는 코스를 내 코스로 가져오거나 GPX 파일로 다운로드합니다.
6. 사용자는 공개 코스나 러닝 경험을 게시글로 공유합니다.
7. 다른 사용자는 게시글에 댓글을 남기고, 코스나 게시글에 좋아요를 누를 수 있습니다.
8. 사용자는 부적절한 코스, 게시글, 댓글을 신고할 수 있습니다.

## MVP 범위

- 지도 위 경로 직접 그리기
- 코스 저장
- 코스 수정
- 코스 목록 조회
- 코스 목록 이름 검색, 태그 필터, 기본 정렬
- 지도 범위 기반 코스 탐색
- 코스 상세 조회
- 공개 코스 내 코스로 가져오기
- GPX 다운로드
- 사용자 계정 및 공개 프로필
- 게시글 작성, 조회, 수정, 삭제
- 게시글에 코스 첨부
- 댓글 작성, 조회, 수정, 삭제
- 코스와 게시글 좋아요
- 좋아요 수 기준 인기순 정렬
- 코스, 게시글, 댓글 신고

## MVP 제외 범위

- 실시간 러닝 기록
- 속도, 페이스, 칼로리 기록 관리
- 팔로우/팔로워
- DM 또는 실시간 채팅
- 푸시 알림
- 추천 알고리즘
- 기간별 랭킹
- 유료 결제
- 웨어러블 연동

## MVP 이후 후보

아래 항목은 데모 버전에서 필요성이 확인되었지만, MVP 공통 계약에는 포함하지 않습니다.
구현 시점이 정해지면 이 저장소에서 데이터 모델과 API 계약을 먼저 확정합니다.

### 코스 작성 경험 개선

- 코스 즐겨찾기
- 지도 위 포인트 번호 표시
- 개별 포인트 삭제
- 포인트 드래그 이동
- 포인트 간 거리 또는 누적 거리 표시
- 루프 코스 닫기 제안
- 좌표 수가 많을 때 경로 단순화 제안

### 제품 품질

- 접근성 라벨 및 스크린리더 대응
- 다국어 지원을 위한 문자열 외부화
- 다크 모드
- 외부 API 실패 재시도
- CI에서 타입 검사와 테스트 자동 실행
- 오류 및 크래시 모니터링

### MVP 이후 소셜 확장

실시간 러닝 추적, 러닝 세션 기록, 주간/월간 통계, 칼로리, 음성 안내, 웨어러블 연동은
Runvas의 코스 설계/공유 MVP와 다른 제품 축입니다.

팔로우, DM, 푸시 알림, 추천 알고리즘, 기간별 랭킹, 운영자용 모더레이션 콘솔은
커뮤니티 MVP 이후 확장으로 분리합니다.

## 프로젝트별 책임

### runvas-mobile

- 지도 UI 표시
- 경로 그리기 인터랙션
- 코스 생성/수정 화면
- 코스 탐색 화면
- 커뮤니티 게시글 목록/상세 화면
- 댓글 작성 UI
- 좋아요 및 신고 UI
- GPX 다운로드 요청 및 파일 처리
- 서버 API 호출

### runvas-backend

- 코스 저장 및 조회 API
- 좌표 데이터 검증
- 지도 범위 기반 검색
- GPX 응답 생성
- 사용자 인증
- 게시글, 댓글, 좋아요, 신고 API
- 작성자 권한 검증
- 페이지네이션과 인기순 정렬
- 데이터 영속화

### runvas-docs

- 공통 API 계약
- 공통 데이터 모델
- 좌표 및 단위 규칙
- GPX 생성 규칙
- 백엔드/모바일 간 합의가 필요한 기술 기준
